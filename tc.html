Frequently Asked Questions

What kind of patterns can it detect?

The logic is designed to detect several common patterns: nested loops ($O(n¬≤)$, $O(n¬≥)$), efficient sorting algorithms ($O(n \log n)$), binary search and other divide-and-conquer methods ($O(\log n)$), single loops ($O(n)$), and constant time operations ($O(1)$).

Is this tool 100% accurate?

While significantly improved, this tool is a high-level estimator for educational purposes. It excels at analyzing imperative code with clear loop structures but may not perfectly analyze complex recursion, algorithm-specific optimizations, or unconventional code patterns.

How does the calculator handle recursion?

Currently, the tool's primary strength is analyzing iterative algorithms (using loops). It does not perform a full recursive analysis, which is significantly more complex. A recursive function might be flagged based on its content (e.g., if it contains loops), but the result may not reflect the overall recursive complexity.

Why is there a button for Space Complexity?

Space complexity is the other critical measure of an algorithm's performance, describing how much memory it uses. The "Space Complexity" button will take you to a dedicated tool designed to analyze memory usage patterns in your code, providing a complete picture of your algorithm's efficiency.
