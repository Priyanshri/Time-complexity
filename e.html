Frequently Asked Questions

What is Time Complexity?

Time complexity is a computational concept that describes the amount of time it takes to run an algorithm. It's usually expressed using Big O notation, which describes the algorithm's performance as the input size grows.

What kind of patterns can it detect?

The logic is designed to detect several common patterns: nested loops ($O(n²)$, $O(n³)$), efficient sorting algorithms ($O(n \log n)$), binary search and other divide-and-conquer methods ($O(\log n)$), single loops ($O(n)$), and constant time operations ($O(1)$).

Is this tool 100% accurate?

While significantly improved, this tool is still a high-level estimator for educational purposes. It excels at analyzing imperative code with clear loop structures but may not perfectly analyze complex recursion, algorithm-specific optimizations, or unconventional code patterns.
